Media Servers: The Basics

A media server is a network device which acts as a central location for users to access media files on a network. There are three main components to any media server system:

  1. Server – this is a smart device which ‘serves’ clients with the requested media files.
  2. Storage Device – which is used to store all the media files such as videos, photos, computer files.
  3. Clients – these are devices that the end user can access the media from. Examples: TV, mobile phone, your personal computer.
  4. Software – a media server needs to have appropriate software that is configured to allow the aforementioned components to communicate with the with the server and access files.

The above illustration depicts a basic Media Server network setup. Media Server Networks range in size and complexity depending on host of factors. The remainder of this article focuses on building and configuring a Home Media Server for personal use. This guide is just that, a guide that is designed to assist in the initial setup and configuration of a Home Media Server and not a comprehensive manual on media servers or network configuration. I acknowledge there are additional and more secure ways to configure home networks but they are beyond the scope of this guide. Network Design:

Network planning and design is an iterative process, encompassing topological design, network-synthesis, and network-realization. During this process the goals and purpose of the network are defined as well as the tools or services that will be used to achieve them. I cannot emphasize thie step enough. Following this process will not ensure success but it will prevent a ton of headaches along the way.

  • Goals
      • Create a media center that can be used to store and orgainze digital media
      The media center needs to be able to be accessed both locally and remotely
      Ability to play music or watch videos on networked devices such as phones, smart tvs, laptops etc.
      Download new media through torrents and news servers
      Implement password protection and https
      VPN capabilities

Now that I have a clear set of goals I can begin the process of selecting the tools and services that can be employed to meet them.

  • Ubuntu Server
      • Media will be stored on the server configured for local and remote access via ssh as well as the internet
  • Docker
      • Create virtual containers within Ubuntu to perform network services.
  • Plex Media Server
      • Media player with both local and remote access
      Compatible with most networkable devices
  • Transmoission-VPN
      • Bittorrent client with VPN capabilities for downloading torrents
      Accessible remotely and can be configured to work with other services
  • NZBget
      • Search and Download media from multiple newsservers
      Automated features include compiling media files and sorting completed downloads to appropiate folders
      API access and highly configurable
  • Ronarr
      • Personal Video Recorder with IMDB intergration that manages movie library
      Searches for torrents and nzb files
      Automates the process when intergraed with Transmission-VPN and NZBget
  • Sonarr
      • Personal Video Recorder with IMDB intergration that manages the tvshow library
      Searches for torrents and nzb files
      Automates the process when intergraed with Transmission-VPN and NZBget
  • Lidarr
      • Personal Music Recorder to manage the music library
      Intergrated with the Headphones indexer movie library
      Searches for torrents and nzb files
      Automates the process when intergraed with Transmission-VPN and NZBget
  • Traefik
      • Reverse proxy
      Configured to work with Let's Encrypt so services can be accessed securely through https

These services are all highly configurable and can be intergrated to automate much of the searching, downloading and file sorting processes that can be very time consuming.
